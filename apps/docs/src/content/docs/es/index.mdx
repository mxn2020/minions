## ¿Por qué Minions?

<CardGrid stagger>
  <Card title="Tres primitivas simples" icon="pencil">
    Todo se construye a partir de **Minions** (instancias), **MinionTypes** (esquemas) y **Relations** (enlaces tipados).
  </Card>
  <Card title="Complejidad progresiva" icon="add-document">
    Comienza con un agente plano. Crece hacia estructuras anidadas con memoria, habilidades y equipos — sin necesidad de migración.
  </Card>
  <Card title="Validación basada en esquemas" icon="approve-check">
    Cada campo se valida contra su esquema de tipo. No más incertidumbre sobre la forma de los datos de tu agente.
  </Card>
  <Card title="Agnóstico de framework" icon="setting">
    Funciona con cualquier backend de almacenamiento, cualquier runtime, cualquier framework de IA. TypeScript y Python puros, sin opiniones sobre infraestructura.
  </Card>
</CardGrid>
