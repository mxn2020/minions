## Pourquoi Minions ?

<CardGrid stagger>
  <Card title="Trois primitives simples" icon="pencil">
    Tout est construit à partir de **Minions** (instances), **MinionTypes** (schémas) et **Relations** (liens typés).
  </Card>
  <Card title="Complexité progressive" icon="add-document">
    Commencez avec un agent simple. Évoluez vers des structures imbriquées avec mémoire, compétences et équipes — sans migration nécessaire.
  </Card>
  <Card title="Validation pilotée par schéma" icon="approve-check">
    Chaque champ est validé par rapport à son schéma de type. Plus de doutes sur la forme de vos données d'agent.
  </Card>
  <Card title="Agnostique vis-à-vis du framework" icon="setting">
    Fonctionne avec n'importe quel backend de stockage, n'importe quel runtime, n'importe quel framework IA. TypeScript et Python purs, sans opinions sur l'infrastructure.
  </Card>
</CardGrid>
